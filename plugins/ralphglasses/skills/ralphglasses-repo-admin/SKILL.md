# Ralphglasses Repo Admin

Use this skill for repo readiness, worktree management, and deterministic maintenance tasks that should stay MCP-native instead of shell-first.

## Default workflow

1. Begin with health and schema checks:
   - `ralphglasses_doctor`
   - `ralphglasses_validate`
   - `ralphglasses_config_schema`
2. For new or drifted repos, use `ralphglasses_repo_scaffold` and `ralphglasses_repo_surface_audit`.
3. Manage branch isolation through:
   - `ralphglasses_worktree_list`
   - `ralphglasses_worktree_create`
   - `ralphglasses_worktree_cleanup`
4. Capture support or forensic state with `ralphglasses_debug_bundle`.
5. Export downstream-facing artifacts only when needed:
   - `ralphglasses_telemetry_export`
   - `ralphglasses_theme_export`

## When to use it

- Repo onboarding and validation
- Worktree hygiene
- Config-key lookup and defaults review
- Debug bundle creation for CI or support follow-up

## Guardrails

- Prefer `ralphglasses_repo_surface_audit` over ad hoc manual parity checks.
- Run validation before scaffold changes on an existing repo.
- Use `ralphglasses_repo_optimize` only when the task is explicitly about improving the repo surface, not just reading it.
